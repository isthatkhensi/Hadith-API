API Documentation

The Hadithit Developer API is a web service that provides access to Hadith collections. The API is intended for non-commercial purposes and allows users to retrieve Hadiths by specifying the collection, book name, and Hadith number.

API Endpoint

All API queries require a GET request using a URL of the following format: https://hadithit.com/api/collections/<collection>/<book_name>/<hadith_number>

Where:

  • <collection> is the name of the collection to retrieve hadith from.
  • <book_number> is the number of the book within the collection to retrieve hadith from.
  • <hadith_number> is the number of the hadith within the book to retrieve.

Examples:

  • http://hadithit.com/api/collections/sahih_bukhari/1/1
  • http://hadithit.com/api/collections/sahih_muslim/2/345

Collections

The API supports the following Hadith collections:

  • Sahih Al-Bukhari (bukhari)
  • Sahih Muslim (muslim)
  • Sunan Abi Dawud (abudawud)
  • Jami` at-Tirmidhi (tirmidhi)
  • Sunan an-Nasa'i (nasai)
  • Sunan Ibn Majah (ibnmajah)

Response Format

The API returns a JSON response containing the requested Hadith information. The JSON response includes the Hadith text, narrator, and source.
Examples:
{
"collection": "sahih_bukhari",
"book_number": 1,
"hadith_number": 1,
"text": "Narrated 'Umar bin Al-Khattab: I heard Allah's Messenger (ï·º) saying, "The reward of deeds depends upon the intentions and every person will get the reward according to what he has intended. So whoever emigrated for worldly benefits or for a woman to marry, his emigration was for what he emigrated for."",
"narrator": "Umar bin Al-Khattab",
"source": "Sahih Bukhari"
}

Possible Errors

The API may return errors in the following cases: Invalid collection name Invalid book number Invalid Hadith number In these cases, the API will return a 404 error along with an error message.
